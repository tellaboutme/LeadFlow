# Governance

LeadFlow is currently maintained by `@tellaboutme`.

## Decision making

- product and architecture decisions are recorded in `docs/DECISIONS.md`;
- issues discuss proposed work;
- pull requests contain implementation review;
- security matters use private vulnerability reporting.

## Maintainer responsibilities

- protect `main`;
- review dependency/security alerts;
- keep releases reproducible;
- avoid misleading project claims;
- preserve a respectful contribution environment;
- document breaking changes.

## Solo-maintainer limitation

Required human approval is disabled while there is only one trusted maintainer because pull-request authors cannot approve their own changes. Required CI, manual gates, PR review checklists and squash merges remain enforced.

When another trusted maintainer joins, one approving code-owner review becomes mandatory.
